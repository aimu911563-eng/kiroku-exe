# Inventory Dashboard（在庫管理 v0.1）

・売店舗での在庫管理における「経験頼りの発注」と「在庫過不足」の課題を解決するために開発したWebアプリです。
　売上予測から必要量を自動算出し、現場で即確認できる形にすることで、発注精度の向上と業務効率化を実現しました。

---

## 背景

従来は以下のような課題がありました：

- Excelでの在庫管理が属人化している
- 売上と必要量が紐づいておらず、経験頼りの発注になっている
- 在庫不足や廃棄が発生しやすい
- 現場でリアルタイムに確認できない

これらを解決するため、本システムを開発しました。

---

## 主な機能

- 売上予測に基づいた必要量の自動計算
- 食材ごとのケース換算表示（pack単位対応）
- 天気情報の表示（判断材料として利用）
- iPad常設表示（現場で即確認可能）
- 5分ごとの自動更新＋手動更新ボタン
- 掃除タスク表示＆完了記録機能

---

## 技術構成

- Frontend: React + Vite
- Backend: Cloudflare Workers（Hono）
- Database: Supabase
- Hosting: Cloudflare Pages / Render
- API設計: REST(/api/inventory/viewなど)
- API: Open-Meteo（天気情報）

---

## 工夫した点

- 売上に応じた必要量計算ロジックを実装（per_100kベース）
- ケース単位での表示により、現場の発注単位に合わせた設計
- iPadでの常設表示を想定し、シンプルなUIに最適化
- APIは5分ポーリングにしてリアルタイム性と負荷のバランスを調整

---

## 想定利用環境

- 店舗内iPadに常設表示
- スタッフがリアルタイムで確認
- 発注判断の補助ツールとして使用

---

## バージョンについて

v0.1では「表示」に特化し、現場導入のハードルを下げることを優先しました。
今後は在庫入力や発注機能を追加予定です。

---

## 今後の展望

- 実在庫入力による差分管理
- 発注提案機能
- ロス分析・KPI可視化
- OCRによる納品書読み取り（Domino Insightと連携）
