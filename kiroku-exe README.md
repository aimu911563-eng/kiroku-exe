# Kiroku.exe（有給申請システム）

従業員の有給申請をデジタル化し、申請・承認・管理を一元化するWebアプリです。
従来の口頭・LINE・紙による申請の曖昧さを解消し、
「誰が・いつ・何日分申請したか」を明確に管理できるようにしました。

---

## 背景

従来は以下のような課題がありました：

- 有給申請が口頭やLINEで行われ、記録が残らない
- 承認状況が曖昧でトラブルになりやすい
- 有給残数の管理が煩雑
- 管理者側の確認・対応負担が大きい

これらを解決するため、本システムを開発しました。

---

## 主な機能

- 有給申請（従業員）
- 承認 / 却下（管理者）
- 申請履歴の一覧表示
- 有給残数管理（承認時にのみ減算）
- 却下理由の入力・保存
- メール通知（承認 / 却下）

---

## 業務フロー

1. 従業員が申請
2. 管理者が確認
3. 承認 or 却下
4. 承認時のみ有給残数を減算
5. 結果を従業員へ通知

---

## 技術構成

- Frontend: TypeScript + Vite
- Backend: Cloudflare Workers（Hono）
- Database: Supabase
- 認証: トークンベース認証（自前実装）
- メール: Resend

---

## 工夫した点

- 有給消費は「申請時ではなく承認時」に減算
  → 誤申請や却下時の整合性を保つため

- 承認済 → 却下で残数を元に戻す設計
  → データ整合性を担保

- 却下時は理由入力を必須化
  → 不透明な却下を防ぐ

- UIはスマホ前提で設計
  → 現場での利用を想定

---

## 技術的なポイント

- トークンに employee_id を含め、他人のデータ参照を防止
- 管理者APIはミドルウェアで制御
- 状態遷移（pending → approved / rejected）を明確に管理

---

## 設計思想

本システムでは「業務上の整合性」を最優先に設計しています。

- 有給消費は申請時ではなく承認時に行う
  → 誤申請・却下時のデータ不整合を防ぐため

- 状態遷移を明確に定義（pending / approved / rejected）
  → 状態の曖昧さを排除

- 承認済み→却下時は残数を戻す
  → データの一貫性を担保

単なるCRUDではなく、業務ロジックに基づいた状態管理を意識しています。
